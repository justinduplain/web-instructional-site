2: Review the home page layout and style.

Open index.html in a browser with a web inspector and note the home page layout.

  • There are two columns set up under the hero image
  • Each column has header text, an image, descriptive text and a button
  • There is a .row style set up. Each row is set to be 100% wide and clear all floats.å
  • Images inside of rows are set to be 100% the width of their parent container.
  • There is a two column and three column layout set up.
  • The two-column setup uses the class col-half which creates a div that is 50% of its parent’s div width and is floated left. Each .col-half has 40 px of padding on the left and right, which creates an 80px gutter.
  • The three-column setup uses the class col-third, which creates a div that is 1/3 (33.333%) of its parent’s width and is floated left.
  • You may note that the leading and trailing col-third divs on this page contain a non-breaking space character (html code is "&nbsp;"). This space is added to place content in the column div. Without content, the div would not have any width to create our first and last columns. We add this html code for a space so that we can give form to the column.
  • A .button style is set up to easily create clickable buttons. Note how these buttons have been added in a new row using the two-column layout. This has the effect of keeping the buttons aligned vertically even through the text above them is not.

Note that there is an "after" pseudo class set up for the row class.

.row:after { 
    content: "";
    display: table;
    clear: both; 
}

All you really need to know about this is that this is a workaround to make your layout do what you expect. This setting fixes a zero-height problem that occurs to divs that contain only floated elements. If you want to know more, you can read about it on stack-overflow. The solution is typically called "clearfix" and usually gets it's own style but I've placed it into each row class for ease of use.

Configure a hyperlink for mobile phone use

The home page displays a phone number in the contact information area. Wouldn’t it be handy if a person using a smartphone could tap on the phone number to call the company? You can make that happen by using tel: in a hyperlink. Configure a hyperlink assigned to a class named mobile that contains the phone number as shown below: 

<a class="mobile" href="tel: 888-555-555">888-555-555</a>

But wait a minute, a telephone link could confuse those visiting the site with a desktop browser. Code another phone number directly after the hyperlink. Code a span element assigned to a class named desktop around the phone number as shown below:

<span class="desktop">888-555-5555</span>

Don’t worry about the two phone numbers that are now on the page. You’ll configure CSS to show the appropriate phone number (with or without the telephone link) to your website visitors. Save the index.html file.

8: Add media queries and styles for tablet view. 

The default view for our website is going to be the desktop view, which we've already completed so we don't need to add a media query for desktop. Instead, we'll add media queries to add breakpoints to modify our default layout if the screen is smaller than a normal desktop view. 

Below is an image of how the website will look on a tablet. Basically we'll just be setting the wrapper to 100% of the width of the device (view image in new window).

Code a Tablet Media Query

At the BOTTOM of your CSS file, code a media query for tablet device viewport size. A max width of 1024 will work well with our current layout. 

@media only screen and (max-width: 1024px) {

}

Add the following styles inside of the media query to push the wrapper to the edges of the screen, but without going below 300 pixels wide. 

body {
   margin: 0;
   padding: 0;
}

#wrapper {
  width: 100%;
  min-width: 300px;
  margin: 0;
  padding: 0;
}

Then add some styles to display text better at this screen size. 

main {
   font-size: 1.15em;
}

main h2 {
    font-size: 1.15em;
}

9: Add media queries and styles for mobile view. 

Below are images of how the website will look on mobile. We want a single column layout where each navigation element to display at full width and each of the columns will transition to display at 100% of the page width (open in new window: nav layout image and column layout image).

 

Code a Mobile Media Query

BELOW the tablet media query, code a media query for mobile device viewport size. A max width of 768 pixels is typical.

@media only screen and (max-width: 768px) {

}

Add the following styles inside of your mobile media query: 

First, we'll give the header some extra padding and set the navigation to display correctly.

header {
   padding: 20px;
}

nav, nav ul, nav li {
   padding: 0;
   margin: 0;
}

nav li {
   display: block;
}

nav a {
   display: block;
   padding: .75em
}

Remember those columns we set up? Let's modify those to display correctly in mobile.

In mobile, we'll typically only want to use a single-column layout. The styles below change each column to display at full width with a little bit of padding on the left and right and some vertical margin. 

.col-half {
   float: none;
   width: 100%;
   padding-left: 10px;
   padding-right: 10px;
   margin-bottom: 28px;
 }

.col-third {
   float: none;
   width: 100%;
   padding-left: 10px;
   padding-right: 10px;
 }
